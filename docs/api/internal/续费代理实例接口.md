# 续费代理实例接口

## 基本信息

| 项目           | 说明                 |
|--------------|--------------------|
| 接口地址         | `/api/order/renew` |
| 请求方式         | `POST`             |
| Content-Type | `application/json` |
| 返回格式         | `application/json` |

## 请求参数

请求体为 **JSON 对象**。

### 请求体字段

| 参数名         | 类型            | 必填 | 说明            |
|-------------|---------------|:--:|:--------------|
| payPassword | String        | ✅  | 支付密码（6位数字）    |
| items       | Array<Object> | ✅  | 续费实例信息列表，至少一条 |

### items 数组元素字段

| 参数名        | 类型      | 必填 | 说明                                   |
|------------|---------|:--:|:-------------------------------------|
| instanceNo | String  | ✅  | 平台实例编号                               |
| unit       | Integer | ✅  | 续费周期单位（1=天, 3=月, 4=年）                |
| duration   | Integer | ✅  | 续费周期时长（如 duration=30、unit=1 表示续费30天） |
| cycleTimes | Integer | ✅  | 续费周期次数，**由前端根据下方规则计算后传入**            |

### cycleTimes 计算规则

`cycleTimes` 的值取决于实例的 `duration` + `unit` 组合，以及用户在弹窗中选择的续费月数。

| duration | unit | 含义       | cycleTimes 计算方式              | 示例                          |
|----------|------|----------|--------------------------------|-----------------------------|
| 1        | 1    | 按天计费     | 用户选择的月数 × 30（转换为天数）           | 用户选1个月 → cycleTimes = 30   |
| 1        | 3    | 按月计费     | 用户选择的月数                       | 用户选3个月 → cycleTimes = 3    |
| 30       | 1    | 按30天/月计费 | 用户选择的月数                       | 用户选3个月 → cycleTimes = 3    |
| 1        | 4    | 按年计费     | 固定为 1（仅支持选12个月，1年=1个cycleTimes） | 固定 → cycleTimes = 1         |

> **前端交互说明：**
> - `duration=1 且 unit=4`（按年计费）时，续费月数弹窗中 **1个月、3个月、6个月选项禁用**，仅 **12个月** 可选且默认选中，`cycleTimes` 固定传 `1`。
> - 其他组合下，前端根据用户选择的月数按上表公式计算 `cycleTimes` 后传入。

### 请求示例

```json
{
    "payPassword": "168888",
    "items": [
        {
            "instanceNo": "c_gjhfpkxyqjx2n76",
            "unit": 1,
            "duration": 30,
            "cycleTimes": 2
        }
    ]
}
```

> 支持批量续费，`items` 中可传入多个实例对象。

## 返回数据

### 通用响应结构

| 字段      | 类型      | 说明                          |
|---------|---------|:-----------------------------|
| code    | Integer | 状态码：200=成功，400=参数错误，500=系统错误 |
| message | String  | 提示信息                        |
| data    | Object  | 业务数据                        |

### 成功响应

```json
{
    "code": 200,
    "message": "success",
    "data": {
        "appOrderNo": "R20329587392622171151741888888000",
        "orderNo": "C20260328103256675598",
        "status": 1,
        "amount": 8.50
    }
}
```

### data 字段说明（ProxyRenewResultVO）

| 字段         | 类型         | 说明             |
|------------|------------|:---------------|
| appOrderNo | String     | 渠道商续费订单号       |
| orderNo    | String     | 平台订单号          |
| status     | Integer    | 订单状态：1=待处理     |
| amount     | BigDecimal | 本次续费总金额（单位：元）  |

## 错误响应示例

### 支付密码错误（code: 400）

```json
{
    "code": 400,
    "message": "支付密码错误",
    "data": null
}
```

### 参数校验失败（code: 400）

```json
{
    "code": 400,
    "message": "参数校验失败: payPassword: 支付密码不能为空",
    "data": null
}
```

### 续费失败-第三方返回业务错误（code: 500）

```json
{
    "code": 500,
    "message": "续费失败: 实例不存在或已释放",
    "data": null
}
```

### 续费失败-系统异常（code: 500）

```json
{
    "code": 500,
    "message": "续费代理失败，请稍后重试",
    "data": null
}
```
