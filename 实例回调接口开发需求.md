# 实例回调接口开发需求
## 一、实例回调说明
### 1. 我方回调地址
```
http://112.124.51.99/api/callback/notify
```

### 2. 回调携带参数
实例信息变动时，第三方调用该地址会携带两个参数：
```
http://112.124.51.99/api/callback/notify?type=instance&no=c_gz3h2igp6dz9cq3
```

### 3. 回调接口返回格式
要求我方回调接口返回以下JSON格式数据：
```json
{
    "code": "",
    "msg": ""
}
```

---

## 二、获取实例信息接口
### 1. 请求路径
`/api/open/app/instance/v2`

### 2. 请求参数
| 参数        | 二级参数 | 类型       | 必填 | 说明                 |
|-----------|------|----------|----|--------------------|
| instances | -    | []string | 是  | 供应商实例编号(供应商系统内部唯一) |

### 3. 返回数据
| 参数          | 二级参数 | 类型        | 必填 | 说明                                                                  |
|-------------|------|-----------|----|---------------------------------------------------------------------|
| instanceNo  | -    | string    | 是  | 平台实例编号(渠道商续费和释放操作使用该编号)                                             |
| proxyType   | -    | uint      | 是  | 代理类型 101=静态云平台 102=静态国内家庭 103=静态国外家庭 104=动态国外 105=动态国内 201=whatsapp |
| protocol    | -    | string    | 否  | 协议类型 多个用英文逗号分隔 1=socks5 2=http 3=https 4=ssh                        |
| ip          | -    | string    | 是  | 代理地址 用户实际代理访问使用                                                     |
| port        | -    | uint      | 否  | 代理端口                                                                |
| regionId    | -    | string    | 否  | 区域地址                                                                |
| countryCode | -    | string    | 是  | 国家代码                                                                |
| cityCode    | -    | string    | 是  | 城市代码                                                                |
| useType     | -    | string    | 是  | 使用方式 多个用英文逗号分隔 1=账密 2=ip白名单 默认1                                     |
| username    | -    | string    | 是  | 账户名或uuid 动态代理为平台主账号                                                 |
| pwd         | -    | string    | -  | 代理密码 动态代理不适用                                                        |
| orderNo     | -    | string    | -  | 创建该实例的平台订单号                                                         |
| userExpired | -    | int64     | -  | 到期时间 时间戳秒                                                           |
| flowTotal   | -    | float64   | -  | 总流量 单位MB 如果是ip数方式 表示ip个数                                            |
| flowBalance | -    | float64   | -  | 剩余流量                                                                |
| status      | -    | int8      | -  | 状态 1=待创建 2=创建中 3=运行中 6=已停止 10=关闭 11=释放                              |
| renew       | -    | int8      | -  | 是否自动续费 1 自动续费 该字段已废弃                                                |
| bridges     | -    | []string  | -  | 桥地址列表                                                               |
| openAt      | -    | time.Time | -  | 开通时间                                                                |
| renewAt     | -    | time.Time | -  | 最后成功续费时间                                                            |
| releaseAt   | -    | time.Time | -  | 释放成功时间                                                              |
| productNo   | -    | string    | -  | 产品编号                                                                |
| extendIp    | -    | string    | -  | 扩展地址,部分产品该字段有值                                                      |
| projectId   | -    | string    | -  | 项目code                                                              |

---

## 三、功能开发需求
1. 接收第三方的回调请求，判断类型为`instance`时，调用获取代理实例信息的接口；
2. 获取代理实例信息后，先保存至数据库（插入，存在则更新）；
3. 向第三方返回处理成功/失败的信息；
4. 参考`ProxyCallbackController`中产品变动回调的代码逻辑实现；
5. 在`ProxyCallbackController`类的`handleNotify`方法中，补充实现实例信息变动回调的代码逻辑；
6. 依据实例信息接口，完成**实例实体类、Service接口及实现类、DAO接口及实现类**的开发。
