## 请求路径
`/api/open/app/instance/open/v2`

## 请求参数
### 主接口中的params参数

| 参数         | 二级参数         | 三级参数  | 类型         | 必填 | 说明                                                       |
|------------|--------------|-------|------------|----|----------------------------------------------------------|
| appOrderNo | -            | -     | string(32) | 是  | 渠道商订单号，保持唯一，我方作幂等性检查，同样的订单号不重复处理                         |
| params     | -            | -     | array      | 是  | 购买代理产品列表，最多支持100个                                        |
| -          | productNo    | -     | string     | 是  | 商品编号，推荐✅（按商品编号购买时，后面8项商品筛选条件无意义）→ 着重看下面红色部分备注            |
| -          | proxyType    | -     | uint16     | 否  | (1) 代理类型，详见字典                                            |
| -          | countryCode  | -     | string     | 否  | (2) 国家代码                                                 |
| -          | cityCode     | -     | string     | 否  | (3) 城市代码                                                 |
| -          | supplierCode | -     | string     | 否  | (4) 供应商代码（可为null，随机分配）                                   |
| -          | unit         | -     | int8       | 否  | (5) 单位：1=天 2=周(7天) 3=月(自然月) 4=年(自然年365, 366) 10=无限制      |

| -          | productType  | -     | int        | 否  | (7) 产品定义的类型：1=独享 2=共享                                    |
| -          | count        | -     | int        | 是  | 购买数量（实例个数），静态必填，默认1，一次最大20                               |
| -          | cycleTimes   | -     | int32      | 是  | 购买时长周期数，此字段对有时长的产品有意义，默认1表示产品的`duration*unit`的时长，详见 → 备注 |

| -          | cidrBlocks   | -     | array      | 否  | 支持网段及数量                                                  |

## 备注
- `cycleTimes` 为新增字段，以区分产品的`duration`字段，老版本使用的`duration`会自动兼容，新的对接方式请使用`cycleTimes`字段
- 产品购买分为：
    1.  按产品编号（`productNo`）唯一指定的产品
    2.  按产品筛选条件（`proxyType`/`countryCode`/`cityCode`/`supplierCode`/`ispType`/`duration`/`unit`）购买
- 当产品编号存在时，产品筛选条件不起作用（推荐按产品编号购买，这是唯一的、价格确定的方式，若存在号段也可指定购买的号段）
- 按筛选条件购买时，筛选条件越多，产品越少，易因找不到产品而购买失败，且不能号段筛选（不建议用此形式）
- 购买数量（`count`）和购买周期数（`cycleTimes`）必填，不填默认为1，即购买一条一个时间周期
- 产品列表的时长：`duration` 和单位 `unit` 的结合，代表这款产品的最小时长

#### eg
- `duration=1` `unit=1`：最少按1天卖，`cycleTimes=1` 表示购买1天，`cycleTimes=30` 代表30天
- `duration=1` `unit=3`：最少按1月卖，`cycleTimes=1` 表示购买1月，`cycleTimes=12` 代表12月（3代表自然月，按购买当月天数计算，如2月28/29天、3月31天等）
- `duration=30` `unit=1`：最少按30天卖，`cycleTimes=1` 表示购买30天，`cycleTimes=12` 代表360天
- `duration=1` `unit=4`：最少按1年卖，`cycleTimes=1` 表示购买1年（365或366天）

## 费用计算
### 静态产品的费用
```
产品价格 * cycleTimes * count + 带宽价格 * extBandWidth * cycleTimes * count + 使用桥费用 * cycleTimes * count
```

- **带宽价格**
    - 必须产品支持，`extBandWidth=0` 表示默认带宽，无费用
- **桥（高速通道）**
    - 必须实名认证企业可用
    - 若 `useBridge=1` 不会产生桥费用；若不传，跟随app配置，app配置为使用桥时，购买海外产品（含香港、台湾）会使用桥
    - 桥使用需要先获取桥地址（动态）

### 流量产品的费用
```
产品价格 (MB) * flow
```

- 流量产品 `count` 不起作用
- 流量产品只对 `flow` 起作用
